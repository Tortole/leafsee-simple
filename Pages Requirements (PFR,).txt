Pages functionality requirements (PFR)
PFR-B Base
    PFR-B-1. Base page
        PFR-B-1.1. Базовая страница является оберткой с элементами, которые отображаются на всех страницах в дополнение к основному контенту страницы
        PFR-B-1.2. Базовая страница должна состоять из
            1. блока с поисковой строкой
            2. блока с боковым меню со ссылками на другие страницы сайта
            3. блока с основным контентом
    PFR-B-2. Блок с поисковой строкой
        PFR-B-2.1. Данный блок должен иметь следующие элементы
            1. поисковая строка
            2. кнопка бокового меню
            3. название и логотип сайта
            4. изображение аватара пользователя
            5. иконка уведомления
            6. кнопка для входа
            7. кнопка для регистрации
        PFR-B-2.2. Поисковая строка должна
            1. иметь следующие элементы
                1. поле для ввода текста
                2. кнопка для запуска поиска
            2. перенаправлять на страницу поиска и передавать текст для поиска из поля ввода при нажатии одного из перечисленных
                1. кнопки для запуска поиска (PFR-B-2.2.1.2)
                2. клавиши Enter
        PFR-B-2.3. Нажатие на кнопку бокового меню должно
            1. скрывать боковое меню, если оно раскрыто
            2. раскрывать боковое меню, если оно скрыто
        PFR-B-2.4. Нажатие на название и логотип сайта должно переносить на главную страницу
        PFR-B-2.5. Нажатие на изображение аватара пользователя должно раскрывать меню со списком следующих элементов:
            1. секция, которая при нажатии переносит в личный кабинет (PFR-PA-1)
            2. секция, которая при нажатии переносит на страницу с видео пользователя (PFR-PA-2)
            3. секция, которая при нажатии позволяет пользователю выйти из аккаунта
        PFR-B-2.6. Нажатие на иконку уведомления должно раскрывать меню, которое
            1. показывает список уведомлений с их описанием при наличии уведомлений
            2. сообщать пользователю, что уведомлений нет при отсутствии уведомлений
        PFR-B-2.7. Нажатие на кнопку для входа (PFR-B-2.1.6) должно вызывать блок для входа (PFR-A-1)
        PFR-B-2.8. Нажатие на кнопку для регистрации (PFR-B-2.1.7) должно вызывать блок для регистрации (PFR-A-2)
        PFR-B-2.9. Если пользователь
            1. вошёл в аккаунт, то
                1. должны отображаться
                    1. изображение аватара пользователя (PFR-B-2.1.4)
                    2. иконка уведомления (PFR-B-2.1.5)
                2. должны быть скрыты
                    1. кнопка для входа (PFR-B-2.1.6)
                    2. кнопка для регистрации (PFR-B-2.1.7)
            2. не вошёл в аккаунт, то
                1. должны отображаться
                    1. кнопка для входа (PFR-B-2.1.6)
                    2. кнопка для регистрации (PFR-B-2.1.7)
                2. должны быть скрыты
                    1. изображение аватара пользователя (PFR-B-2.1.4)
                    2. иконка уведомления (PFR-B-2.1.5)
    PFR-B-3. Блок с боковым меню
        PFR-B-3.1. При нажатии на кнопку бокового меню в блоке с поисковой строкой, блок бокового меню должен иметь следующие поведения, в зависимости от начального состояния
            1. если изначально раскрыт, то при нажатии кнопки бокового меню
                1. боковое меню скрывается, если раскрыто, и расширяет блок с основным контентом на освободившееся место
                2. боковое меню раскрывается, если скрыто, и сужает блок с основным контентом на ширину самого бокового меню
            2. если изначально скрыто, то при нажатии кнопки бокового меню
                1. боковое меню раскрывается, если скрыто, затемняя тёмно-прозрачным цветом остальную область на странице
                2. боковое меню скрывается, если раскрыто, убирая затемнение на остальной области на странице
        PFR-B-3.2. По умолчанию на всех страницах данный блок должен быть развёрнут, если не указано другое
        PFR-B-3.3. Боковое меню должно содержать в виде списка следующие элементы:
            1. секция, которая при нажатии переносит на главную страницу
            2. секция, которая при нажатии переносит на страницу с видео из подписок пользователя
            3. секция, которая при нажатии переносит на страницу с историей просмотренных видео
            4. секция, которая при нажатии переносит на страницу с понравившемся видео
            5. секция с подписками пользователя
        PFR-B-3.4. Секция с подписками пользователя
            1. должна состоять из
                1. списка подписок
                2. кнопка для раскрытия полного списка подписок
            2. в списке подписок (PFR-B-3.4.1.1) изначально должны отображаться первые 5 подписок, по дате подписки, остальные должны быть скрыты
            3. при нажатии на кнопку для раскрытия полного списка подписок (PFR-B-3.4.1.2) в списке подписок (PFR-B-3.4.1.1) должны отобразиться остальные подписки
            4. при повторном нажатии на кнопку для раскрытия полного списка подписок (PFR-B-3.4.1.2) список подписок (PFR-B-3.4.1.1) должен сокращаться до исходного состояния
    PFR-B-4. Блок с основным контентом расположенный в оставшемся пространстве
        PFR-B-4.1. Данный блок должен отображать основное наполнение страницы
PFR-A Authentication blocks
    PFR-A-1. Log in block
        PFR-A-1.1. При вызове данного блока, он должен появиться на странице, на которой его вызвали и затемнить остальные элементы
        PFR-A-1.2. На данном блоке должны быть
            1. поле для ввода логина или почты
            2. поле для ввода пароля
            3. кнопка для входа
            4. кнопка для вызова блока регистрации
        PFR-A-1.3. Формат заполнения полей (PFR-A-1.2.1, PFR-A-1.2.2) должен соответствовать таковым требованиям к соответствующим полям модели пользователя (!!!)
        PFR-A-1.4. Текстовое поле для ввода пароля (PFR-A-1.2.2) должно маскировать вводимый текст, заменяя символы на символ звёздочки «*»
        PFR-A-1.5. У текстового поля для ввода пароля (PFR-A-1.2.2) должна быть кнопка для показа замаскированного текста (PFR-A-1.4)
        PFR-A-1.6. При нажатии кнопки для входа (PFR-A-1.2.3)
            1. пользователь должен входить в аккаунт, которому соответствует введённый в поле логин или почта (PFR-A-1.2.1), если все условия ниже выполняются
                1. поля для логина или почты (PFR-A-1.2.1), а также для пароля (PFR-A-1.2.2) заполнены
                2. такой аккаунт существует
                3. введённый в поле пароль (PFR-A-1.2.2) соответствует таковому на аккаунте
            2. должно выводиться сообщение о незаполненных полях и указывать какие именно поля незаполненны, если хотя бы одно из полей не заполнено
            3. если указанного аккаунта не существует или введены неправильные данные, то
                1. должно выводиться сообщение вида «неправильный логин и/или пароль»
                2. текст в поле для ввода логина или почты (PFR-A-1.2.1) оставить как ввёл пользователь
                3. очистить поле для ввода пароля (PFR-A-1.2.2)
        PFR-A-1.7. После успешного входа
            1. блок для входа должен скрыться
            2. текущая страница должна перезагрузиться
        PFR-A-1.8. При нажатии на кнопку для вызова блока регистрации (PFR-A-1.2.4)
            1. блок для входа должен скрыться
            2. блок для регистрации должен появиться (PFR-A-2)
    PFR-A-2. Register block 
        PFR-A-2.1. При вызове данного блока, он должен появиться на странице, на которой его вызвали и затемнить остальные элементы
        PFR-A-2.2. На блоке для регистрации должны быть
            1. обязательные для заполнения пользователем поля для
                1. юзернейма
                2. пароля
                3. повторного ввода пароля
            2. необязательные для заполнения пользователем поля для
                1. электронной почты
                2. никнейма
                3. фамилии
                4. имени
            3. кнопка для регистрации
            4. кнопка для перехода на страницу входа
        PFR-A-2.3. Формат заполнения полей (PFR-A-2.2.1, PFR-A-2.2.2) должен соответствовать таковым требованиям к соответствующим полям модели пользователя (!!!), при это поля необязательные для заполнения (PFR-A-2.2.2) могут быть пустыми
        PFR-A-2.4. Пустые поля должны сохраняться в базу данных как Null
        PFR-A-2.5. Для поля юзернейма (PFR-A-2.2.1.1) добавить надпись, что после регистрации юзернейм изменить нельзя
        PFR-A-2.6. Текстовые поля для ввода пароля (PFR-A-2.2.1.2, PFR-A-2.2.1.3) должны маскировать вводимый текст, заменяя символы на символ звёздочки «*»
        PFR-A-2.7. У текстовых полей для ввода пароля (PFR-A-2.2.1.2, PFR-A-2.2.1.3) должна быть кнопка для показа замаскированного текста (PFR-A-2.6)
        PFR-A-2.8. При нажатии на кнопку регистрации (PFR-A-2.2.3)
            1. должен создаваться новый аккаунт по введённый данным и пользователь должен сразу входить в этот аккаунт, если все условия ниже выполняются
                1. обязательные для заполнения поля (PFR-A-2.2.1) заполнены
                2. формат данных в полях (PFR-A-2.2.1, PFR-A-2.2.2) соответствует требованию о формате данных (PFR-A-2.3)
                3. значения полей пароля (PFR-A-2.2.1.2) и повторного ввода пароля (PFR-A-2.2.1.3) совпадают
                4. не существует другого аккаунта с такими же юзернеймом (PFR-A-2.2.1.1) и адресом электронной почты (PFR-A-2.2.2.1)
            2. должно выводиться сообщение о незаполненных обязательных для заполнения полях (PFR-A-2.2.1) и указывать какие именно поля незаполненны, если хотя бы одно из этих полей не заполнено
            3. должно выводиться сообщение о неправильно заполненных данных и указывать какие именно поля заполнены неправильно, если данные в полях (PFR-A-2.2.1, PFR-A-2.2.2) не соответствуют формату (PFR-A-2.3)
            4. должно выводиться сообщение о несовпадении значений в полях пароля (PFR-A-2.2.1.2) и повторного ввода пароля (PFR-A-2.2.1.3)
            5. должно обобщённое сообщение о уже существующем аккаунте с такими же данными, если юзернейм (PFR-A-2.2.1.1) или адрес электронной почты (PFR-A-2.2.2.1) введённые пользователем аналогичны таковым в другом уже созданном аккаунте
        PFR-A-2.9. После успешной регистрации и входа
            1. блок для регистрации должен скрыться
            2. текущая страница должна перезагрузиться
        PFR-A-2.10. При нажатии на кнопку для перехода на блок входа (PFR-A-2.2.4)
            1. блок для регистрации должен скрыться
            2. блок для входа должен появиться (PFR-A-1)
PFR-V Videos pages
    PFR-V-1. Main page
        PFR-V-1.1. По умолчанию, при переходе по адресу сайта без указания конкретной страницы, должна загружаться данная страница
        PFR-V-1.2. На главной странице должны отображаться рекомендательные видео в виде отдельных блоков
        PFR-V-1.3. Блок с рекомендательным видео (PFR-V-1.2) должен состоять из следующих элементов
            PFR-V-1.3.1. превью видео
            PFR-V-1.3.2. название видео
            PFR-V-1.3.3. никнейм автора
            PFR-V-1.3.4. юзернейм автора
            PFR-V-1.3.5. аватар автора
            PFR-V-1.3.6. количество просмотров
            PFR-V-1.3.7. дата публикации
        PFR-V-1.4. При нажатии на блок с рекомендательным видео (PFR-V-1.2) пользователя должно переносить
            PFR-V-1.4.1. на страницу автора видео, если нажатие было на никнейм автора (PFR-V-1.3.3) или аватар автора (PFR-V-1.3.5)
            PFR-V-1.4.2. на страницу с воспроизведением самого видео, если нажатие было на  остальные элементы
    PFR-V-2. Video player page
        PFR-V-2.1. На данной странице боковое меню (PFR-B-3) должно быть изначально скрыто
        PFR-V-2.2. Данная страница должная содержать следующие элементы
            PFR-V-2.2.1. видеоплейер
            PFR-V-2.2.2. описание к видео
            PFR-V-2.2.3. блок комментариев
            PFR-V-2.2.4. блок с рекомендательными видео
        PFR-V-2.3. Видеоплейер (PFR-V-2.2.1)
            PFR-V-2.3.1. Должен воспроизводить видео, на которое перешёл пользователь
            PFR-V-2.3.2. Должен иметь следующие элементы
                1. кнопка паузы и продолжения воспроизведения
                2. кнопка включения и выключения звука
                3. ползунок регулирования уровня громкости
                4. кнопка для перехода в полноэкранный режим или выхода из него
                5. подменю с настройками воспроизведения видео, содержащее
                    1. кнопку для изменения качества видео
                    2. кнопку для изменения скорости воспроизведения
                6. кнопка открытия и скрытия подменю настроек
                7. текущее время видео, прошедшее с начала воспроизведения
                8. общее время видео
                9. ползунок перемотки видео
            PFR-V-2.3.3. Ползунок видео (PFR-V-2.3.2.9) должен
                1. отображать текущее время видео на прогресс баре
                2. позволять переходить на другие временные моменты видео
        PFR-V-2.4. Описание к видео (PFR-V-2.2.2)
            PFR-V-2.4.1. Должно иметь следующие элементы
                1. название видео
                2. аватар автора видео
                3. никнейм автора видео 
                4. количество подписчиков у автора видео
                5. кнопка, позволяющая пользователю подписаться на автора видео
                6. кнопка, позволяющая поставить лайк видео
                7. кнопка, позволяющая поставить дизлайк видео
                8. счётчик лайков и дизлайков
                9. количество просмотров видео
                10. дата публикации видео
                11. теги видео
                12. текстовое описание к видео
        PFR-V-2.5. Блок комментариев (PFR-V-2.2.3)
            PFR-V-2.5.1. Должен содержать
                1. общее количество комментарии
                2. кнопка для переключения порядка сортировки комментариев
                    1. новые (по-умолчанию)
                    2. популярные (по количеству лайков)
                3. блок для создания нового комментария от имени пользователя
                    1. текстовое поле
                    2. кнопка отмены комментария (очистить текстовое поле (PFR-V-2.5.1.3.1))
                    3. кнопка публикации комментариев
                4. список комментариев от других людей
            PFR-V-2.5.2. Каждый отдельный комментарий должен состоять из следующих элементов
                1. аватар автора комментария
                2. никнейм автора комментария
                3. юзернейм автора комментария
                4. дата публикации комментария
                5. текст комментария
                6. кнопка, позволяющая поставить лайк комментарию
                7. кнопка, позволяющая поставить дизлайк комментарию
                8. количество лайков
                9. количество дизлайков
                10. кнопки, позволяющая написать новый ответный комментарий на текущий комментарий от имени авторизированного пользователя
                11. кнопки для раскрытия или сокрытия списка ответных комментариев на текущий комментарий
                12. надпись о изменении комментария
                13. если комментарий принадлежит авторизованному в данный момент пользователю, то дополнительно должны отображаться следующие элементы
                    1. кнопка для открытия подменю управления комментарием, содержащая следующие элементы
                        1. кнопка для удаления комментария
                        2. кнопка для изменения комментария
            PFR-V-2.5.3. При нажатии на кнопку для создания ответного комментария (PFR-V-2.5.2.10) должен появляться блок, аналогичный блоку для создания нового комментария (PFR-V-2.5.1.3)
            PFR-V-2.5.4. Если у комментария отсутствуют ответные комментарии, то кнопка для раскрытия ответов (PFR-V-2.5.2.11) должна быть скрыта
            PFR-V-2.5.5. При нажатии на кнопку для раскрытия ответов (PFR-V-2.5.2.11) под соответствующим комментарием должен появиться список ответных комментариев
            PFR-V-2.5.6. Надпись о изменении комментария (PFR-V-2.5.2.12) должна быть
                1. скрыта, если комментарий не изменялся с момента публикации
                2. видна, если комментария был изменён с момента публикации
            PFR-V-2.5.7. При нажатии на кнопку изменения комментарий (PFR-V-2.5.2.13.1.2) на месте комментария должны появиться следующие элементы
                1. текстовое поле, содержащая текст исходного комментария
                2. кнопка для отмены изменений
                3. кнопка для сохранения изменений
